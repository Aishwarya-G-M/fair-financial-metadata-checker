# FAIR Financial Metadata Checker

A small FastAPI service that evaluates metadata for controlled-access financial datasets using a FAIR-inspired scoring model.

The project is designed as an MVP for checking whether financial dataset metadata is sufficiently documented to support discoverability, access governance, interoperability, and reuse.

## Why this project

Financial datasets are rarely fully open, but they still benefit from high-quality metadata.

This project applies FAIR-inspired checks to datasets such as:
- cross-border payments
- fraud detection data
- AML monitoring datasets
- risk analytics datasets

The goal is not to determine whether a dataset is public, but whether its metadata is documented well enough to support responsible discovery, controlled access, and reuse.

## Current FAIR mapping

| Principle | Current checks |
|---|---|
| Findable | identifier, dataset name, description |
| Accessible | access rights, access request contact |
| Interoperable | schema version, domain standard |
| Reusable | usage terms, provenance, retention policy |

## Setup

Create and activate a virtual environment, then install dependencies.

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Run locally

Start the API with:

```bash
python -m uvicorn app.main:app --reload
```

Then open:

- `http://127.0.0.1:8000/health`
- `http://127.0.0.1:8000/docs`

## Run tests

```bash
pytest
```

## Example request

```json
{
  "identifier": "urn:wise:cross-border-payments:v1",
  "dataset_name": "Cross-Border Payments Analytics Dataset",
  "description": "Aggregated dataset of international money transfer events used for fraud monitoring and corridor analysis.",
  "owner_team": "Payments Risk Engineering",
  "data_domain": "cross_border_payments",
  "access_rights": "restricted",
  "access_request_contact": "data-governance@company.com",
  "license_or_usage_terms": "Internal use only for approved analytics and risk investigations.",
  "provenance": "Derived from payment events generated by the transfer orchestration platform.",
  "schema_version": "1.0.0",
  "domain_standard": "internal-payments-metadata-v1",
  "contains_pii": true,
  "retention_policy": "Retain for 5 years, then review for deletion."
}
```

## Example response shape

The `/assess` endpoint returns:

- overall score
- number of passed checks
- grouped scores for F, A, I, and R
- individual check results
- recommendations for missing metadata

## Current status

This is an MVP and currently uses a lightweight rules-based scoring model.

Possible next steps:
- richer FAIR checks
- weighted scoring
- better OpenAPI examples
- CI test workflow
- domain-specific metadata profiles for payments, fraud, or AML
